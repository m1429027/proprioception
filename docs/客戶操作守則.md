# 肩關節雷射投影量測系統操作守則

## 文件目的

本文件提供客戶端實際操作本系統時的完整流程說明。  
內容以「第一次使用也能照著完成」為原則，從設備準備、程式啟動、校正、量測、練習、考試、存檔，到結束收尾都依序整理。

本系統的主要目的，是將肩關節活動路徑量測、分段練習與考試記錄半自動化，降低人工標記造成的誤差與操作負擔。

---

## 一、系統用途簡述

本系統可完成以下工作：

- 使用相機偵測受試者操作的紅色雷射點
- 將雷射點位置對應到投影畫面
- 建立一條從 `START` 到 `END` 的正式參考路徑
- 將參考路徑依比例切成數個分段
- 讓受試者依路徑進行練習
- 讓受試者依指定目標分點進行考試
- 記錄每個考試回合的移動路徑、時間與誤差
- 匯出 `Path JSON` 與 `Exam Excel`

---

## 二、操作前準備

### 1. 硬體準備

請先確認以下設備皆已連接並可正常使用：

- 電腦
- 相機
- 投影機
- 投影幕或投影平面
- 紅色雷射筆

### 2. 環境準備

請盡量確保：

- 投影範圍內沒有多餘的高亮反光物
- 背景不要有容易干擾的紅色物件
- 相機可以完整看見投影區域
- 投影畫面清楚可見
- 雷射點打到投影區時，能被相機穩定拍到

### 3. 軟體啟動

進入專案資料夾後啟動程式：

```powershell
python Demo.py
```

若使用 `uv`（Python 環境工具）：

```powershell
uv run python Demo.py
```

啟動後會出現兩個主要畫面：

- `Control Panel`：操作者使用
- `Projector Screen`：投影給受試者使用

## 三、畫面說明

## 1. Control Panel

`Control Panel` 主要由操作者使用，包含：

- 相機畫面
- 四個模式分頁
- 目前模式的操作按鈕
- 狀態提示訊息
- 相機設定面板
- 受試者資料夾資訊
- 最新 `Exam` 回合誤差顯示

## 2. Projector Screen

`Projector Screen` 主要由受試者觀看，包含：

- 比例尺
- 十字準星
- `START`
- `END`
- 路徑線
- 分段標記
- 練習或考試時的參考路徑高亮區段

## 四、四個模式說明

本系統共有四個模式：

- `Initial`
- `Measure`
- `Practice`
- `Exam`

### 1. Initial

用途：

- 設定受試者資料夾
- 相機設定
- 比例尺設定
- 校正投影座標
- 全螢幕或無邊框設定

可用按鈕：

- `Calibrate`
- `Settings`
- `Scale`
- `Full`
- `Border`
- `Subject`

### 2. Measure

用途：

- 建立正式的量測參考路徑

可用按鈕：

- `Reset Path`
- `Split`
- `Save Path`

### 3. Practice

用途：

- 依照已量測好的參考路徑做分段練習

可用按鈕：

- `Range`

### 4. Exam

用途：

- 依照指定分點進行考試
- 記錄每個回合的路徑、時間與誤差

可用按鈕：

- `Range`
- `Reset Trial`
- `Save Exam`

## 五、完整使用流程

以下建議作為每一位受試者的標準操作流程。

### 步驟 1：啟動程式

啟動後先確認：

- 相機畫面正常
- 投影畫面正常
- 雷射點能在相機畫面中被看到

若沒有看到相機畫面或畫面異常，請先不要進行後續步驟。

### 步驟 2：進入 Initial 模式

程式啟動後，先停留在 `Initial` 模式完成前置設定。

### 步驟 3：設定受試者資料夾

按下 `Subject`。

此步驟的目的：

- 為本次受試者建立專屬資料夾
- 後續 `Path` 與 `Exam Excel` 都會存到這個資料夾中

操作方式：

1. 若已有受試者資料夾，可選擇既有資料夾
2. 若要建立新受試者資料夾，可建立新的資料夾名稱

注意事項：

- 沒有先設定 `Subject Folder`，不能進入 `Measure`、`Practice`、`Exam`
- 若建立新資料夾時名稱重複，系統會跳出警告，需重新命名

### 步驟 4：調整相機設定

若紅點偵測不穩，按下 `Settings` 開啟相機設定面板。

可調整參數：

- `Exposure`
- `Focus`
- `Min Bright`
- `Min Area`
- `Ignore Bot`

調整建議：

- 雷射點太暗：可微調曝光
- 雷射點不夠清楚：調整對焦
- 雜訊太多：提高 `Min Bright` 或 `Min Area`
- 畫面底部有干擾：提高 `Ignore Bot`

調整時建議一次只改一項，避免不知道哪一項造成影響。

### 步驟 5：設定比例尺

按下 `Scale`。

輸入內容為：

- `50` 個投影像素實際代表幾公分

這會影響：

- 距離換算
- `Exam` 誤差的 `cm` 計算

若比例尺變更，系統會要求後續重新 `Save Path`。

### 步驟 6：進行校正

按下 `Calibrate`。

校正目的：

- 建立相機畫面與投影畫面的座標對應

校正前請確認：

- 4 個 `ArUco` 標記都能被相機看到
- 相機畫面中不要被手、雷射筆或其他物件遮住標記

校正成功後，系統會更新投影對位。

若校正失敗，請檢查：

- 標記是否完整出現在畫面中
- 畫面是否過亮或過暗
- 相機是否失焦

## 六、Measure 模式操作

### 1. 進入 Measure

在完成 `Subject` 設定後，切換到 `Measure`。

### 2. 設定分段數

按 `Split`。

輸入：

- 這條路徑要切成幾段

例如：

- 輸入 `3`，系統會分成 3 段
- 輸入 `4`，系統會分成 4 段

這個分段設定會影響：

- `Practice`
- `Exam`
- 目標分點計算

### 3. 開始量測 path

`Measure` 模式下使用 `Space`：

- 第一次按 `Space`：標記 `START`
- 第二次按 `Space`：標記 `END`

操作流程：

1. 將紅點放到起始位置
2. 按一次 `Space`
3. 受試者沿著實際動作移動紅點
4. 到達最大活動位置後，再按一次 `Space`

系統會在 `START` 與 `END` 之間自動記錄紅點移動路徑。

### 4. 路徑處理方式

目前系統處理路徑的邏輯為：

1. 記錄紅點的實際移動點列
2. 去除連續重複點
3. 排除極端跳點
4. 對中間路徑做較保守的平滑
5. 保留原始 `START` 與 `END`

目的是讓路徑盡量貼近實際移動軌跡，而不是過度簡化成捷徑。

### 5. 重做 path

若本次量測不好，按 `Reset Path` 可重新量測。

`Reset Path` 會清除：

- `START`
- `END`
- 原始路徑
- 處理後路徑

### 6. 儲存 path

量測完成後，按 `Save Path`。

系統會輸出：

- `YYYYMMDD_HHMMSS_path.json`

存放位置：

- 目前受試者資料夾

注意：

- 未先 `Save Path`，不能進入 `Practice` 或 `Exam`
- 若量測後又修改 `Split` 或 `Scale`，需重新 `Save Path`

---

## 七、Practice 模式操作

### 1. 進入 Practice

條件：

- 必須已完成 `Measure`
- 必須已按 `Save Path`

### 2. 顯示整段或部分路徑

按 `Range`。

輸入內容為：

- 從 `START` 顯示到第幾個分點

例如：

- `1`：顯示到第一個分點
- `2`：顯示到第二個分點
- `3`：顯示到第三個分點

`Practice` 模式下，投影畫面會顯示：

- 參考路徑
- 指定分段的高亮區域

用途：

- 讓受試者先熟悉某一段或整段動作路徑

---

## 八、Exam 模式操作

## 1. 進入 Exam

條件：

- 必須已完成 `Measure`
- 必須已按 `Save Path`

進入 `Exam` 後，系統會先詢問：

- 本次每個 `segment` 要做幾個 `trial`

例如：

- `3`
- `5`

代表之後每一個目標分點，都要做相同數量的回合。

## 2. 選擇考試目標 segment

按 `Range`。

輸入：

- 本次要考到第幾個分點

例如：

- `1`：考第一個分點
- `2`：考第二個分點
- `3`：考第三個分點

注意：

- 沒有先選 `Range`，不能開始本次 `segment`

## 3. 每個 trial 的三段式操作

每一個 `trial` 都要按三次 `Space`：

- 第一次：標記 `start`
- 第二次：標記 `target`
- 第三次：標記 `end`

這代表一個完整 `trial` 為：

- 從起點出發
- 到指定目標分點
- 再回到起點

## 4. 每個 trial 完成後顯示的資訊

每完成一個 `trial`，`Control Panel` 會立即顯示：

- `start error`
- `target error`
- `end error`

單位為：

- `cm`

計算方式：

- `start error`：本次 `trial start` 與參考 `path start` 的距離
- `target error`：本次 `trial target` 與參考 `path` 指定分點的距離
- `end error`：本次 `trial end` 與參考 `path start` 的距離

也就是：

- `end` 是拿來比較有沒有回到原點，不是比較 `path end`

## 5. 重做上一個 trial

若上一個已完成回合不理想，按 `Reset Trial`。

目前邏輯：

- 會重做上一個已完成但不滿意的回合
- 不會錯誤清到下一個尚未開始的回合

## 6. 完成一個 segment 後

做完目前 `segment` 的全部 `trial` 後：

- 本 `segment` 資料會先保留在記憶體中
- 不會立刻寫成 `Excel`
- 接著可再按 `Range` 選下一個 `segment`

例如：

1. 做完 `Segment 1`
2. 再選 `Segment 2`
3. 再做 `Segment 2` 的全部 `trial`
4. 之後再做 `Segment 3`

## 7. Save Exam

當所有需要的 `segment` 都做完後，按 `Save Exam`。

系統會輸出：

- `YYYYMMDD_HHMMSS_data.xlsx`

存放位置：

- 目前受試者資料夾

---

## 九、Exam Excel 輸出格式

目前 `Exam Excel` 採用多 `sheet` 結構：

- `Segment 1`
- `Segment 2`
- `Segment 3`
- `Path`

### 1. Segment 工作表

每個 `segment` 一個獨立工作表。

內容包含：

- 每個 `trial` 的 `x`
- 每個 `trial` 的 `y`
- 每個 `trial` 的 `time`
- 每個 `trial` 的 `event`
  - `start`
  - `target`
  - `end`
- 每個 `trial` 的：
  - `start point`
  - `target point`
  - `end point`
  - `start error`
  - `target error`
  - `end error`
  - `total time`

### 2. Path 工作表

最後一個 `Path` 工作表用來保存本次參考路徑。

內容包含：

- `path x`
- `path y`

此路徑為本次 `Exam` 使用的參考 `path`。

## 十、整體操作順序建議

建議每位受試者都照以下順序進行：

1. 啟動程式
2. `Initial`
3. 設定 `Subject`
4. 視需要調整 `Settings`
5. 設定 `Scale`
6. `Calibrate`
7. `Measure`
8. 設 `Split`
9. 量測 `START -> END`
10. `Save Path`
11. `Practice`
12. 設 `Range`
13. 受試者練習
14. `Exam`
15. 設定 `trial` 數
16. 設 `Range`
17. 完成該 `segment` 的全部 `trial`
18. 如需更多目標分點，再選下一個 `Range`
19. 所有 `segment` 完成後按 `Save Exam`
20. 確認檔案已存到受試者資料夾

---

## 十一、常見狀況與處理方式

### 1. 不能進入 Measure / Practice / Exam

請先確認：

- 是否已設定 `Subject Folder`

### 2. 不能進入 Practice / Exam

請先確認：

- 是否已完成量測
- 是否已按 `Save Path`

### 3. Range 設好後仍不能開始 Exam

請確認：

- 是否真的已完成 `Save Path`
- 是否目前還卡在未完成的前一個 `segment`

### 4. 紅點偵測不穩

可先檢查：

- 相機焦距
- 曝光
- 背景反光
- 畫面中是否有其他紅色或高亮干擾

### 5. 校正失敗

請確認：

- 4 個 `ArUco` 是否都在畫面內
- 標記是否清楚
- 相機是否失焦

### 6. Path 看起來不合理

請檢查：

- 量測過程中紅點是否穩定
- 是否中途有大跳點
- 若不理想，按 `Reset Path` 重新量測

### 7. Exam 無法存檔

請確認：

- 所有當前 `segment` 的 `trial` 是否已完成
- 是否還停留在未完成的中途回合
- 是否最後有按 `Save Exam`

### 8. 中途直接關閉程式的風險

目前：

- `Path` 只要已按 `Save Path`，檔案就會保留
- `Exam` 必須按 `Save Exam` 才會正式輸出

因此若 `Exam` 做到一半就直接關閉程式：

- 尚未 `Save Exam` 的資料可能遺失

建議：

- 完成所有需要的 `segment` 後立刻 `Save Exam`

---

## 十二、結束前確認事項

在關閉程式前，請確認：

- 本次受試者的 `path.json` 已成功輸出
- 本次受試者的 `data.xlsx` 已成功輸出
- 檔案位於正確的受試者資料夾中
- 若有多個 `segment`，`Excel` 內已正確出現多個工作表

確認無誤後，再按 `Q` 或直接正常結束程式。
